# Skill: Content Draft

Draft content in the user's brand voice for any format they produce.

## Before You Start

Read [Voice Guide](../../docs/voice-guide.md). This is not optional — it is the source of truth for tone, style, audience, and what to avoid. Do not proceed without reading it.

Also read [Content Types](../../docs/content-types.md) to match the correct format structure.

## Steps

1. **Read the docs** — voice-guide.md first, then the relevant format section in content-types.md
2. **Clarify if needed** — if the brief is ambiguous (unclear topic, unclear format, unclear audience), ask one focused question before drafting. Not five questions. One.
3. **Draft the full piece** — not an outline, not a summary, not bullet points. The actual content, ready to publish or lightly edit.
4. **Apply the voice** — check against the avoid-list in voice-guide.md before finishing
5. **Close with a check-in** — end your response with: "Does this match the voice? Anything to adjust?"

## What NOT to Do

- Do not start with "Great question!" or any filler affirmation
- Do not write in a different format than what was asked for
- Do not produce an outline when a draft was requested
- Do not use words from the avoid-list in voice-guide.md
- Do not skip the voice-guide read even for "just a quick draft"

## If the Voice Guide Isn't Filled In

Tell the user: "Your voice-guide.md has placeholder text. Fill in the [BRACKETED SECTIONS] so I can match your actual voice — otherwise I'll write something generic."

## Format Handling

- If the user doesn't specify a format, check content-types.md for the most common format and use that, but confirm: "I'll draft this as a [format] — is that right?"
- If the requested format isn't in content-types.md, draft in a reasonable format and ask them to add the format to content-types.md for next time
