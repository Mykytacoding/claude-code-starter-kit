# Skill: Daily Review

Run a morning check-in: review what's on deck, surface anything blocking progress, and set up for a productive day.

## Before You Start

Read [Workflows](./docs/workflows.md) to understand how this person organizes their work — their tools, their priorities system, and what a "productive day" means for them.

If `docs/workflows.md` doesn't exist or has placeholder text, tell the user: "Fill in docs/workflows.md first — I need to know how you track work before I can run a useful review."

## Steps

1. **Read workflows.md** — understand their task management setup and what to check
2. **Ask what to review** — if they didn't specify, ask: "Want me to pull from [their tool], or walk through what's on your mind?"
3. **Surface the short list** — identify the 3 most important things to move today, based on urgency and impact
4. **Flag blockers** — anything waiting on someone else, overdue, or unclear
5. **Suggest a starting point** — recommend which item to tackle first and why
6. **End with a check-in question** — "Anything else on your plate I should know about?"

## Output Format

Keep it short. The review should take 2 minutes to read, not 10.

```
## Today's Focus
1. [Most important task] — [why it's first]
2. [Second task]
3. [Third task]

## Blockers / Waiting On
- [Item] — waiting on [person/thing] since [date]

## Suggested Start
[Single sentence: what to open first and why]
```

## What NOT to Do

- Don't list everything in their backlog — just the 3-5 that matter today
- Don't ask clarifying questions about each item — surface them, then ask once at the end
- Don't run this without reading workflows.md first

## Adapting to Different Systems

- **Notion / Linear / Trello:** Ask user to paste in their open items, or use MCP if connected
- **Google Sheets:** Ask for a quick summary of what's open, or use Sheets MCP if connected
- **No tool / mental load:** Ask "What's the 3 things you need to get done today?" and work from there
